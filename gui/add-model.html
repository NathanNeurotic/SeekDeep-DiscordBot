Where does this model live?

The first two stay on your machine and need no API key. The bottom three call out to a remote provider — prompts leave the local machine, so they carry a privacy notice. You can mix any of these per role.

HF
Hugging Face local · no key
Snapshot-download into the local cache. Anything published on huggingface.co. The default for chat, image, and vision roles.
OL
Ollama local daemon · no key
Pulls a tag via POST /api/pull. Needs the Ollama daemon listening on :11434.
API
OpenAI-compat BYK · remote
DeepSeek, OpenAI, Groq, OpenRouter, xAI Grok — anything that speaks the OpenAI Chat Completions shape.
⚠ prompts leave the local machine
ANT
Anthropic BYK · remote
Native /v1/messages with x-api-key + anthropic-version headers.
⚠ prompts leave the local machine
GMI
Gemini BYK · remote
Native :generateContent with x-goog-api-key.
⚠ prompts leave the local machine

Where does it answer from?

Base URL and key for the remote provider. Stored in your local .env via POST /config — never logged, never leaves your machine except to authenticate this provider over TLS.

⚠
Privacy disclosure. Prompts sent to this backend will leave your local machine. The remote provider receives the full prompt body in cleartext over TLS. Keep the key in .env only; never paste it into chat or commit it.

Defaults vary by provider. DeepSeek = https://api.deepseek.com/v1 · Groq = https://api.groq.com/openai/v1 · Anthropic = https://api.anthropic.com/v1/messages · Gemini = https://generativelanguage.googleapis.com/v1beta.
