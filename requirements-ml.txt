# SeekDeep · ML deps — heavy libraries for /chat (local), /image, /vision,
# and the warmup_local_cache.py model puller.
#
# Installed on first use via the in-app "Install ML libraries" prompt
# (POST /deps/install endpoint). NOT installed by the Tauri sidecar's
# first-run flow — the boot deps in requirements-local.txt are enough to
# bring the server up; ML deps are lazy because their install is ~2 GB
# and ~5-10 minutes, which is a bad first-launch experience for users
# who only want to play with the GUI / remote backends.
#
# Remote-only setups (openai-compat / anthropic / gemini) never need this
# file — the /chat handler routes to HTTP for those backends without ever
# importing torch / transformers / diffusers.

# PyTorch's CUDA wheels live behind a separate index. We previously
# used --extra-index-url which is permissive (pip can resolve from
# EITHER PyPI or the extra URL) and the resolver kept preferring the
# plain PyPI torch over torch+cu128 — users ended up with a CPU-only
# wheel even though they explicitly asked for ML deps. Pin the local
# version (+cu128) so pip MUST pull from pytorch.org. The trio is
# version-locked together by PyTorch so we pin the same triple.
#
# When PyTorch ships a newer toolkit (cu130+), bump these three lines
# and the cu128 URL. The wizard's "wrong wheel" FIX path uses
# pip_install_torch_variant() which handles cu118/cu121/cu124/cu126/cu128
# at runtime — this file just sets the default for fresh installs.
--extra-index-url https://download.pytorch.org/whl/cu128
# Verified-available on https://download.pytorch.org/whl/cu128/ as of 2026-05:
#   torch       2.11.0+cu128  (latest cu128 — cp310..cp314)
#   torchvision 0.26.0+cu128  (matches torch 2.11)
# torchaudio is INTENTIONALLY OMITTED — SeekDeep imports torch +
# torchvision (via diffusers/transformers) but never torchaudio. Pinning
# it caused a ResolutionImpossible: torchaudio 2.10.0+cu128 hard-pins
# torch==2.10.0, which conflicts with our torch 2.11.0 pin. Keep it out
# until SeekDeep actually grows an audio surface.
torch==2.11.0+cu128 ; platform_system == "Windows"
torchvision==0.26.0+cu128 ; platform_system == "Windows"
torch ; platform_system != "Windows"
torchvision ; platform_system != "Windows"
